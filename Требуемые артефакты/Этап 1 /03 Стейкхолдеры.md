## 3. Стейкхолдеры

| Стейкхолдер          | Приоритет | Интересы                                                   | Опасения                                    |
| -------------------- | --------- | ---------------------------------------------------------- | ------------------------------------------- |
| Руководство компании | Высокий   | Рост вовлечённости, узнаваемость бренда, увеличение продаж | Окупаемость инвестиций, репутационные риски |
| Маркетинг            | Средний   | Аналитика, персонализация, эксперименты и промоакции       | Качество и доступность данных               |
| Пользователи         | Высокий   | Мотивация, удобство, безопасность данных                   | Утечки данных, нестабильность сервиса       |
| Партнёры             | Средний   | Интеграции, совместные активности, доступ к аудитории      | Нестабильность API, частые изменения        |
| Команды разработки   | Высокий   | Гибкая архитектура, технологическая свобода                | Сложность системы, технический долг         |
| DevOps / SRE         | Высокий   | Надёжность, масштабируемость, автоматизация                | Рост операционной сложности                 |
| Security / Legal     | Высокий   | Соответствие требованиям защиты данных                     | Штрафы, нарушения регуляций                 |
