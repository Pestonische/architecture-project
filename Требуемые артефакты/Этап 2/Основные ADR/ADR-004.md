# ADR-004: Интеграция с магазином через API и события

## Контекст
В компании уже существуют полноценные торговые приложения, реализующие:
- каталог товаров;
- ценообразование;
- заказы и платежи.

Спортивное приложение должно стимулировать продажи, но не дублировать e-commerce домен.

## Решение
Интегрировать спортивное приложение с магазином через:
- API (каталоги, промоакции);
- события (кампании, рекомендации),
без реализации собственной коммерческой логики.

## Рассмотренные альтернативы
- Встроенный e-commerce внутри приложения

## Причины отказа от альтернатив
- дублирование бизнес-логики;
- рост сложности и ответственности;
- ухудшение пользовательского опыта.

## Последствия
**Положительные:**
- чёткое разделение доменов;
- снижение рисков;
- гибкость развития обеих систем.

**Отрицательные:**
- зависимость от SLA торговых сервисов;
- необходимость контрактного управления API.
